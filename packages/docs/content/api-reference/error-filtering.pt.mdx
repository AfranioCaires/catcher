---
title: Filtragem de Erros
description: Capture erros específicos e deixe os outros quebrarem a aplicação.
---

Às vezes, você só deseja capturar tipos específicos de erros, como erros de validação, enquanto deixa erros críticos (como falhas de conexão com o banco de dados) subirem e quebrarem a aplicação.

## Usando Filtros

Você pode fornecer um array de classes Error como um argumento secundário. Se o erro lançado for uma instância de qualquer classe no array, ele será capturado. Caso contrário, ele será lançado novamente.

<CodeBlock
  filename="error-filter.ts"
  language="typescript"
  code={`import { catchErrorSync } from '@catcherjs/core';

class DatabaseError extends Error {
name = 'DatabaseError';
}

class ValidationError extends Error {
name = 'ValidationError';
}

function processUserData() {
const result = catchErrorSync(
() => {
throw new DatabaseError('Connection failed');
},
[ValidationError]
);

    return result;

}`}
/>

## Exceções Não Capturadas

No exemplo acima, o `DatabaseError` não está na lista de filtros `[ValidationError]`. Portanto, o `catchErrorSync` não retornará um objeto Result; em vez disso, ele lançará novamente o `DatabaseError` e interromperá o programa, o que geralmente é o comportamento desejado para erros fatais.

## Combinando Filtros

Você pode passar várias classes para lidar com diferentes modos de falha esperados simultaneamente.

<CodeBlock
  filename="combined-filters.ts"
  language="typescript"
  code={`import { catchErrorSync } from '@catcherjs/core';

const [error, data] = catchErrorSync(
() => performComplexTask(),
[ValidationError, DatabaseError, NetworkError]
);

if (error) {
console.log(error.name);
}`}
/>
