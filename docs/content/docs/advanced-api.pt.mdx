# API Avançada

O objeto `Result` fornece uma API fluente e rica fortemente inspirada em programação funcional. Esses métodos permitem encadear operações, transformar dados e disparar efeitos colaterais (side-effects) sem quebrar o fluxo de controle.

## Métodos de Transformação

### `map`

Transforma os dados de sucesso usando uma função fornecida. Se o resultado for um erro, ele ignora a função e retorna o erro como está.

```typescript
import { catchErrorSync } from 'catch-error-lib';

const result = catchErrorSync(() => JSON.parse('{"count": 5}'));

// result é Result<{count: number}, Error>
const doubled = result.map(data => data.count * 2);

console.log(doubled.unwrap()); // 10
```

### `mapErr`

Transforma o erro usando uma função fornecida. Se o resultado for um sucesso, ele retorna o sucesso como está.

```typescript
const result = catchErrorSync(() => { throw new Error('Não encontrado') });

const customErrorResult = result.mapErr(err => new CustomError(err.message));
```

## Métodos de Encadeamento

### `andThen` / `flatMap`

Encadeia outra operação que também retorna um `Result`. Se a primeira operação falhar, a operação encadeada é ignorada. Ambos `andThen` e `flatMap` são aliases idênticos.

```typescript
const processData = (raw: string) => {
  return catchErrorSync(() => JSON.parse(raw))
    .andThen(parsed => catchErrorSync(() => validateSchema(parsed)));
};
```

### `orElse`

Recupera-se de um erro retornando um novo `Result`. Se o resultado original for um sucesso, ele ignora a função e retorna o sucesso como está.

```typescript
const result = catchErrorSync(() => fetchFromCache())
  .orElse(() => catchErrorSync(() => fetchFromDatabase()));
```

## Métodos de Side-Effect

### `tap`

Executa um efeito colateral (como um log) se o resultado for um sucesso. Ele não modifica o `Result` e o retorna inalterado, permitindo continuar o encadeamento.

```typescript
const result = catchErrorSync(() => computeValue())
  .tap(data => console.log('Computado com sucesso:', data))
  .map(data => data * 2);
```

### `tapErr`

Executa um efeito colateral se o resultado for um erro.

```typescript
const result = catchErrorSync(() => riskyOperation())
  .tapErr(err => console.error('Operação falhou:', err.message));
```

## Métodos Utilitários

### `unwrapErr`

Retorna o erro contido se o resultado for uma falha. Lança os dados contidos se for um sucesso (o oposto de `unwrap`).

### `toPromise`

Converte o `Result` de volta para uma Promise padrão. Resolve com os dados em caso de sucesso, ou rejeita com o erro em caso de falha.

```typescript
const result = catchErrorSync(() => 42);
const value = await result.toPromise();
```
