# Utilitários e Combinadores

A biblioteca expõe várias funções utilitárias independentes além dos "catchers" principais. Essas funções ajudam a agregar múltiplos Results ou a adaptar estruturas externas para Results.

## Combinadores

Os combinadores ajudam a gerenciar arrays de objetos `Result`.

### `combine`

Combina um array de Results em um único Result. Se todos tiverem sucesso, ele retorna `ok` com um array contendo os dados. Se **qualquer** Result for uma falha, ele retorna imediatamente o **primeiro** erro encontrado.

```typescript
import { combine, ok, err } from 'catch-error-lib';

const combined = combine([ok(1), ok(2), ok(3)]);
console.log(combined.unwrap()); // [1, 2, 3]

const withError = combine([ok(1), err('Falhou'), ok(3)]);
console.log(withError.unwrapErr()); // 'Falhou'
```

### `combineAll`

Semelhante ao `combine`, mas não sofre "curto-circuito" (short-circuit). Ele coleta **todos** os erros. Se todos tiverem sucesso, retorna `ok(dataArray)`. Se um ou mais falharem, retorna `err(errorArray)`.

```typescript
import { combineAll, ok, err } from 'catch-error-lib';

const result = combineAll([ok(1), err('Err1'), err('Err2')]);

if (result.isErr()) {
  console.log(result.error); // ['Err1', 'Err2']
}
```

### `partition`

Divide um array de Results em dois arrays separados: um para os valores de sucesso e um para os erros. Esta função nunca falha.

```typescript
import { partition, ok, err } from 'catch-error-lib';

const { ok: data, err: errors } = partition([ok(1), err('Falha'), ok(3)]);

console.log(data); // [1, 3]
console.log(errors); // ['Falha']
```

## Adaptadores

Os adaptadores convertem outros padrões de programação para o formato `Result`.

### `fromNullable`

Converte um valor anulável (`null` ou `undefined`) em um Result. Você deve fornecer o valor do erro a ser usado caso os dados estejam ausentes.

```typescript
import { fromNullable } from 'catch-error-lib';

const user = null;
const result = fromNullable(user, new Error('Usuário não encontrado'));

console.log(result.isErr()); // true
```

### `fromThrowable`

Envolve uma função síncrona em uma que retorna um `Result` em vez de lançar uma exceção. Isso é útil para criar variantes seguras de funções de terceiros.

```typescript
import { fromThrowable } from 'catch-error-lib';

const safeParse = fromThrowable(JSON.parse);

const result = safeParse('{ invalid }');
```

### `fromPromise`

Um alias para `catchError`. Ele envolve uma Promise e retorna um `AsyncResult`.

```typescript
import { fromPromise } from 'catch-error-lib';

const result = await fromPromise(fetch('...'));
```
