# Utilities & Combinators

The library exposes several standalone utility functions in addition to the core catchers. These functions help you aggregate multiple Results or adapt external structures into Results.

## Combinators

Combinators help you manage arrays of `Result` objects.

### `combine`

Combines an array of Results into a single Result. If all succeed, it returns `ok` with an array of the data. If **any** Result is a failure, it immediately returns the **first** error encountered.

```typescript
import { combine, ok, err } from 'catch-error-lib';

const combined = combine([ok(1), ok(2), ok(3)]);
console.log(combined.unwrap()); // [1, 2, 3]

const withError = combine([ok(1), err('Failed'), ok(3)]);
console.log(withError.unwrapErr()); // 'Failed'
```

### `combineAll`

Similar to `combine`, but it does not short-circuit. It collects **all** errors. If all succeed, it returns `ok(dataArray)`. If one or more fail, it returns `err(errorArray)`.

```typescript
import { combineAll, ok, err } from 'catch-error-lib';

const result = combineAll([ok(1), err('Err1'), err('Err2')]);

if (result.isErr()) {
  console.log(result.error); // ['Err1', 'Err2']
}
```

### `partition`

Splits an array of Results into two separate arrays: one for the successful values and one for the errors. It never fails.

```typescript
import { partition, ok, err } from 'catch-error-lib';

const { ok: data, err: errors } = partition([ok(1), err('Fail'), ok(3)]);

console.log(data); // [1, 3]
console.log(errors); // ['Fail']
```

## Adapters

Adapters convert other programming patterns into the `Result` format.

### `fromNullable`

Converts a nullable value (`null` or `undefined`) into a Result. You must provide the error value to use in case the data is missing.

```typescript
import { fromNullable } from 'catch-error-lib';

const user = null;
const result = fromNullable(user, new Error('User not found'));

console.log(result.isErr()); // true
```

### `fromThrowable`

Wraps a synchronous function into one that returns a `Result` instead of throwing. This is useful for creating safe variants of third-party functions.

```typescript
import { fromThrowable } from 'catch-error-lib';

const safeParse = fromThrowable(JSON.parse);

const result = safeParse('{ invalid }');
```

### `fromPromise`

An alias for `catchError`. It wraps a Promise and returns an `AsyncResult`.

```typescript
import { fromPromise } from 'catch-error-lib';

const result = await fromPromise(fetch('...'));
```
